## Overview

Before attempting a Windows 10 to 11 upgrade, the migration tool validates that the target system meets all Windows 11 hardware requirements using the WhyNotWin11 compatibility checker.

**Source:** `~/workspace/source/roles/fase_1/tasks/main.yml:108-121`

## Validation Process

The compatibility check is performed in the FASE 1 role using the following workflow:

<Steps>
  <Step title="Execute WhyNotWin11">
    Runs `WhyNotWin11.exe /e CSV /s` in silent mode to generate a CSV report
  </Step>
  <Step title="Parse CSV Results">
    Reads the CSV file and extracts 11 boolean values (True/False) for each requirement
  </Step>
  <Step title="Set Ansible Facts">
    Creates individual Ansible facts for each compatibility check
  </Step>
  <Step title="Evaluate Compatibility">
    Sets `compatible: true` only if ALL 11 requirements return 'True'
  </Step>
</Steps>

## Windows 11 Hardware Requirements

The following table lists all 11 requirements checked by the migration tool, extracted from the debug output in the source code.

| Requirement | Ansible Fact | Required Value | Description |
|-------------|--------------|----------------|-------------|
| **Architecture** | `architecture` | 64 Bit CPU, 64 Bit OS | System must have 64-bit processor and operating system |
| **Boot Method** | `boot_method` | UEFI | System must boot using UEFI (not legacy BIOS) |
| **CPU Compatibility** | `cpu_compatibility` | Intel Core 8th Gen+ or AMD Ryzen 2000+ | CPU must be Intel Core (i3, i5, i7, i9) 8th generation or newer, OR AMD Ryzen (3, 5, 7, 9, Threadripper) 2000 series or newer |
| **CPU Core Count** | `cpu_core_count` | 2 or more | Processor must have at least 2 cores |
| **CPU Frequency** | `cpu_frequency` | 1 GHz or more | Processor clock speed must be at least 1 GHz |
| **DirectX + WDDM2** | `directx_wddm2` | DirectX 12+ & WDDM2 2.x | Graphics must support DirectX 12 and Windows Display Driver Model 2.x |
| **Disk Partition Type** | `disk_partition_type` | GPT | System disk must use GPT partition scheme (not MBR) |
| **RAM Installed** | `ram_installed` | 4 GB or more | System must have at least 4 GB of RAM |
| **Secure Boot** | `secure_boot` | Enabled | Secure Boot must be enabled in UEFI firmware |
| **Storage Available** | `storage_available` | 64 GB or more [C:] | System drive (C:) must have at least 64 GB free space |
| **TPM Version** | `tpm_version` | 2.0 (Enabled) | Trusted Platform Module 2.0 must be present and enabled |

<Note>
  All 11 requirements must return 'True' for the system to be considered compatible. If even one requirement fails, the upgrade will not proceed (when using the correct `compatible` condition instead of `not compatible`).
</Note>

## CSV Parsing Logic

**Source:** `~/workspace/source/roles/fase_1/tasks/main.yml:93-106`

The CSV file generated by WhyNotWin11 contains a header row and a data row. Each compatibility check is parsed from the second line using comma-split indexing:

```yaml
- name: 2.9 Analizar la Compatibilidad del PC
  ansible.builtin.set_fact:
    architecture: "{{ csv_content.stdout_lines[1].split(',')[1] }}"
    boot_method: "{{ csv_content.stdout_lines[1].split(',')[2] }}"
    cpu_compatibility: "{{ csv_content.stdout_lines[1].split(',')[3] }}"
    cpu_core_count: "{{ csv_content.stdout_lines[1].split(',')[4] }}"
    cpu_frequency: "{{ csv_content.stdout_lines[1].split(',')[5] }}"
    directx_wddm2: "{{ csv_content.stdout_lines[1].split(',')[6] }}"
    disk_partition_type: "{{ csv_content.stdout_lines[1].split(',')[7] }}"
    ram_installed: "{{ csv_content.stdout_lines[1].split(',')[8] }}"
    secure_boot: "{{ csv_content.stdout_lines[1].split(',')[9] }}"
    storage_available: "{{ csv_content.stdout_lines[1].split(',')[10] }}"
    tmp_version: "{{ csv_content.stdout_lines[1].split(',')[11] }}"
```

## Compatible Boolean Logic

**Source:** `~/workspace/source/roles/fase_1/tasks/main.yml:123-141`

The `compatible` variable is set through a two-step process:

### Step 1: Initialize to False

```yaml
- name: 2.11 Valor Booleano de compatible
  ansible.builtin.set_fact:
    compatible: false
```

By default, all systems are assumed to be incompatible.

### Step 2: Set to True if All Requirements Pass

```yaml
- name: 2.12 Validar si el PC es Compatible con Windows 11
  ansible.builtin.set_fact:
    compatible: true
  when:
    - architecture == 'True'
    - boot_method == 'True'
    - cpu_compatibility == 'True'
    - cpu_core_count == 'True'
    - cpu_frequency == 'True'
    - directx_wddm2 == 'True'
    - disk_partition_type == 'True'
    - ram_installed == 'True'
    - secure_boot == 'True'
    - storage_available == 'True'
    - tpm_version == 'True'
```

<Warning>
  The `when` condition uses a typo: `tmp_version` in line 105 but `tpm_version` in line 141. This inconsistency should be corrected to use `tpm_version` throughout (Trusted Platform Module, not "tmp").
</Warning>

The `compatible` boolean is only set to `true` when ALL 11 conditions evaluate to the string `'True'` (case-sensitive). This ensures:

- **Single point of failure:** If any requirement fails, the system remains incompatible
- **Explicit validation:** All requirements must be explicitly met
- **Safe by default:** Systems are incompatible unless proven otherwise

## Compatibility Reporting

**Source:** `~/workspace/source/roles/fase_1/tasks/main.yml:154-186`

After validation, the role generates a detailed compatibility report saved to the Ansible controller:

```yaml
- name: 2.14 Generar reporte de compatibilidad
  ansible.builtin.copy:
    content: |
      Reporte de Compatibilidad para: {{ ansible_fqdn }}
      -----------------------------------------------
      - Architecture: {{ architecture }}
      - Boot Method: {{ boot_method }}
      - CPU Compatibility: {{ cpu_compatibility }}
      - CPU Core Count: {{ cpu_core_count }}
      - CPU Frequency: {{ cpu_frequency }}
      - DirectX + WDDM2: {{ directx_wddm2 }}
      - Disk Partition: {{ disk_partition_type }}
      - Ram Installed: {{ ram_installed }}
      - Secure Boot: {{ secure_boot }}
      - Storage Available: {{ storage_available }}
      - TPM Version: {{ tmp_version }}
    dest: roles/fase_1/files/{{ ansible_fqdn }}-compatibility_report.txt
    mode: "0644"
```

## Common Compatibility Issues

<AccordionGroup>
  <Accordion title="TPM 2.0 Not Enabled" icon="lock">
    **Symptom:** `tpm_version == 'False'`
    
    **Resolution:**
    1. Enter UEFI/BIOS settings during boot
    2. Navigate to Security settings
    3. Enable TPM/Security Device
    4. Set TPM version to 2.0 (if multiple versions available)
    5. Save and reboot
  </Accordion>

  <Accordion title="Secure Boot Disabled" icon="shield">
    **Symptom:** `secure_boot == 'False'`
    
    **Resolution:**
    1. Enter UEFI settings
    2. Navigate to Boot or Security settings
    3. Enable Secure Boot
    4. Ensure boot mode is set to UEFI (not Legacy/CSM)
    5. Save and reboot
  </Accordion>

  <Accordion title="Legacy BIOS Boot" icon="server">
    **Symptom:** `boot_method == 'False'`
    
    **Resolution:**
    1. Check if system supports UEFI
    2. Convert MBR disk to GPT using `mbr2gpt.exe`
    3. Change BIOS boot mode from Legacy/CSM to UEFI
    4. Verify boot order in UEFI settings
    
    <Warning>
      Converting MBR to GPT requires backup. Test on non-production systems first.
    </Warning>
  </Accordion>

  <Accordion title="MBR Partition Scheme" icon="hard-drive">
    **Symptom:** `disk_partition_type == 'False'`
    
    **Resolution:**
    Use Windows built-in MBR2GPT tool:
    ```powershell
    mbr2gpt /validate /disk:0
    mbr2gpt /convert /disk:0
    ```
    
    This is typically paired with enabling UEFI boot mode.
  </Accordion>

  <Accordion title="Unsupported CPU" icon="microchip">
    **Symptom:** `cpu_compatibility == 'False'`
    
    **Resolution:**
    
    Windows 11 requires:
    - Intel: 8th generation Core or newer (Coffee Lake+)
    - AMD: Ryzen 2000 series or newer (Zen+)
    
    If CPU is too old:
    - System cannot be upgraded to Windows 11
    - Consider hardware replacement
    - Or continue using Windows 10 until 2025 end-of-support
    
    <Info>
      Registry bypasses exist but are not recommended for production environments as they may cause stability issues and won't receive official support.
    </Info>
  </Accordion>
</AccordionGroup>

## WhyNotWin11 Tool

The compatibility checker is an open-source tool from GitHub.

<Card title="WhyNotWin11" icon="github" href="https://github.com/rcmaehl/WhyNotWin11">
  Open-source Windows 11 compatibility checker by rcmaehl
  
  **Usage in playbook:**
  ```yaml
  - name: 2.7 Ejecutar 'WhyNotWin11' (silencioso) y Generar Archivo CSV
    ansible.windows.win_shell: |
      {{ fase_1_dir_whynotwin11 }}/WhyNotWin11.exe /e CSV {{ fase_1_dir_whynotwin11 }}/Salida_Compatibilidad.csv /s
  ```
  
  **Parameters:**
  - `/e CSV` - Export results to CSV format
  - `/s` - Run in silent mode (no GUI)
  - Output file: `Salida_Compatibilidad.csv`
</Card>
